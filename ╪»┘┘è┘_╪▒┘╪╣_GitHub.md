# 📦 دليل رفع ملفات المكتبة على GitHub (خطوة بخطوة)

الهدف: نستضيف الـ13 ملف PDF على GitHub، ونعرضها في المكتبة **بعارض خاص بلا أي زر Google**.

---

## الخطوة 1 — أنشئ حساب GitHub (إن لم يكن عندك)
1. روح إلى **github.com** → **Sign up**
2. أدخل بريدك وكلمة مرور واسم مستخدم (مثلاً `sacsvtprof`)

---

## الخطوة 2 — أنشئ مستودعًا (Repository) عامًّا
1. اضغط زر **+** فوق يمين → **New repository**
2. **Repository name**: `sac-files` (أو أي اسم)
3. اختر **Public** (مهم — لازم عام)
4. فعّل **Add a README file**
5. اضغط **Create repository**

---

## الخطوة 3 — نزّل الملفات الـ13 من Google Drive
افتح كل مجلد Drive ونزّل ملفاته (أو نزّلها كما هي).

## الخطوة 4 — ارفع الملفات إلى المستودع
1. في صفحة المستودع، اضغط **Add file** → **Upload files**
2. اسحب الملفات الـ13 دفعة واحدة (أو على دفعات)
3. انتظر رفعها ثم اضغط **Commit changes**

> ⚠️ ملاحظة: أكبر ملف (الكتاب 3م = 85 ميغا) تحت حد GitHub (100 ميغا) فلا مشكل.

---

## الخطوة 5 — عدّل رابط الملفات في قاعدة البيانات
1. افتح ملف `insert_github.sql`
2. استبدل **USER/REPO** باسم حسابك ومستودعك.
   مثال: لو حسابك `sacsvtprof` والمستودع `sac-files`:
   - من: `cdn.jsdelivr.net/gh/USER/REPO@main/...`
   - إلى: `cdn.jsdelivr.net/gh/sacsvtprof/sac-files@main/...`
   (استعمل خاصية Replace All في أي محرّر نصوص)
3. انسخ الملف كاملاً → Supabase → **SQL Editor** → **Run**

---

## ✅ انتهى!
افتح `library.html` → قسم «وثائق» و«كتب» → اضغط «عرض»:
الملف يُعرض **داخل المكتبة بعارض خاص** (تصفّح صفحات، تكبير، تحميل) — **بلا أي زر Google**.

---

### 💡 ملاحظات
- **jsDelivr** (الرابط) هو CDN مجاني سريع يقرأ من GitHub تلقائيًا.
- لإضافة ملفات جديدة لاحقًا: ارفعها على GitHub بنفس الطريقة، ثم استعمل «لوحة الإدارة» في المكتبة أو أضف سطرًا في SQL.
- إذا لم يظهر ملف جديد فورًا عبر jsDelivr، انتظر بضع دقائق (تحديث الكاش) أو استعمل الرابط `@latest` بدل `@main`.
