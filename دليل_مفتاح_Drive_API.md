# 🔑 دليل الحصول على مفتاح Google Drive API (مجاني — 5 دقائق)

هذا المفتاح يسمح لنا بقراءة أسماء وملفات مجلدات Drive **بدقة 100%**،
ثم نولّد قائمة الـ142 مذكرة تلقائيًا في قاعدة البيانات.

> ملاحظة: المفتاح للقراءة فقط، وآمن. لن نرفع أي ملف — فقط نقرأ الأسماء والروابط.

---

## الخطوة 1 — افتح Google Cloud Console
1. روح إلى **console.cloud.google.com**
2. سجّل الدخول بنفس حساب Google (اللي فيه المذكرات)

---

## الخطوة 2 — أنشئ مشروعًا (Project)
1. فوق يسار، اضغط قائمة المشاريع → **New Project**
2. الاسم: `sac-drive` → **Create**
3. انتظر ثوانٍ ثم اختر المشروع من القائمة

---

## الخطوة 3 — فعّل Google Drive API
1. من القائمة (☰) → **APIs & Services** → **Library**
2. ابحث عن **Google Drive API**
3. اضغط عليها → **Enable**

---

## الخطوة 4 — أنشئ مفتاح API
1. من (☰) → **APIs & Services** → **Credentials**
2. فوق اضغط **+ Create Credentials** → **API key**
3. راح يظهر المفتاح فورًا (يبدأ بـ `AIza...`)
4. **انسخه** واحتفظ به.
5. (اختياري للأمان) اضغط **Edit API key** → تحت **API restrictions** اختر **Restrict key** → فعّل **Google Drive API** فقط → Save

---

## الخطوة 5 — تأكد أن المجلدات عامة
كل مجلدات المذكرات لازم تكون **"أي شخص لديه الرابط – مُشاهِد"** (وهي كذلك أصلًا لأنك شاركتِها).

---

## ✅ بعد الانتهاء:
أعطني **مفتاح API** (يبدأ بـ `AIza...`) وأنا:
1. أقرأ الـ44 مجلد بدقة عبر الـ API
2. أستخرج الـ142 ملف بأسمائها الصحيحة + اسم الأستاذ لكل ملف
3. أولّد لك ملف SQL جاهز (روابط Drive مباشرة) — تلصقه في Supabase
4. تظهر كل المذكرات في المكتبة، مصنّفة حسب الأستاذ ✅

> 🔒 مفتاح Drive API عادي أن يُستخدم في السكريبت. بعد الانتهاء تقدر تحذفه من Credentials إن أردت.
