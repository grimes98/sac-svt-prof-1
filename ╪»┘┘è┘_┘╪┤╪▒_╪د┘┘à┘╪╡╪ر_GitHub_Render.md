# 🚀 دليل نشر منصة SAC · SVT prof على GitHub و Render (مجاناً وخطوة بخطوة)

لقد تم تجهيز مشروعك بالكامل داخل المجلد الرسمي (**`sac-svt-app`**) وتزويده بكافة الواجهات (`index.html`، `login.html`، `admin.html`)، إضافة إلى خادم **Node.js (`server.js`)** وإعدادات النشر المثالية (`package.json`) ليعمل فوراً وسلسة على منصة **Render (`onrender.com`)** وبدون أي أخطاء.

إليك الخطوات التفصيلية لنشر منصتك وجعلها متاحة لجميع الأساتذة عبر رابط مباشر على الإنترنت:

---

## 📦 المرحلة الأولى: رفع ملفات المجلد على GitHub (`github.com`)

### 1. أنشئ مستودعاً جديداً (Repository) على GitHub:
1. اذهب إلى موقع [GitHub.com](https://github.com) وسجلي الدخول بحسابك.
2. اضغطي على علامة **`+`** في أعلى يمين الشاشة واختاري **New repository**.
3. في خانة **Repository name** اكتبي اسماً لمشروعك، مثلاً: `sac-svt-prof-platform` أو `sac-svt-app`.
4. اختاري **Public** (عام) ليكون متاحاً لخدمة النشر.
5. اضغطي على الزر الأخضر **Create repository**.

### 2. ارفع ملفات المجلد (`sac-svt-app`):
يمكنك الرفع بإحدى الطريقتين:

#### 🔹 الطريقة (أ): السحب والإفلات مباشرة من المتصفح (الأسهل بدون برامج):
1. بعد إنشاء المستودع في GitHub، اضغطي على رابط **`uploading an existing file`**.
2. اسحبي جميع الملفات والمجلدات الموجودة داخل مجلد `sac-svt-app` (وهي أساساً: `index.html`، `login.html`، `admin.html`، `server.js`، `package.json`، ومجلد `assets/` وباقي الصفحات) وأفلتها في نافذة الرفع.
3. انتظري انتهاء الرفع بالكامل، ثم اضغطي في الأسفل على زر **Commit changes**.

#### 🔹 الطريقة (ب): باستخدام أوامر Git (عبر موجه الأوامر Terminal):
افتحي التيرمينال داخل مجلد `sac-svt-app` ونفّذي الأوامر التالية بالترتيب (مع استبدال رابط مستودعك):
```bash
git init
git add .
git commit -m "النسخة الشاملة لمنصة SAC · SVT prof مع صفحة الدخول ولوحة الأدمين وخادم النشر"
git branch -M main
git remote add origin https://github.com/اسم_حسابك/sac-svt-app.git
git push -u origin main
```

## 🌐 المرحلة الثانية: ربط GitHub بـ Render ونشر المنصة (`onrender.com`)

بعد أن أصبحت الملفات على GitHub، انتقلي إلى منصة [Render.com](https://render.com) لنشر الموقع مجاناً في دقيقة واحدة:

### 1. التسجيل والدخول في Render:
1. ادخلي إلى موقع [Render.com](https://render.com) واضغطي على **Get Started** أو **Sign in**.
2. اختاري التسجيل باستخدام حساب غيت هاب **Sign in with GitHub** لكي يتمكن Render من قراءة مستودعاتك تلقائياً.

### 2. إنشاء خدمة ويب جديدة (Web Service):
1. من لوحة تحكم Render الرئيسية (Dashboard)، اضغطي على زر **`New +`** واختاري **`Web Service`**.
2. ستظهر لك قائمة بمستودعاتك على GitHub، ابحثي عن مستودع مشروعك (`sac-svt-app`) واضغطي بجانبه على زر **Connect**.

### 3. ضبط إعدادات النشر (مهم جداً وسهل):
ستفتح لك صفحة إعدادات الخدمة، أدرجي الإعدادات كالتالي بالضبط:
- **Name:** اكتبي اسم الرابط الذي تريدينه لمنصتك، مثلاً: `sac-svt-prof` (سيكون الرابط النهائي `https://sac-svt-prof.onrender.com`).
- **Region:** اختاري أقرب منطقة (مثلاً `Frankfurt` أو `Paris` أو `Oregon`).
- **Branch:** اختاري `main` (أو `master`).
- **Runtime / Environment:** اختاري **`Node`**.
- **Build Command:** اكتبي: `npm install` (أو اتركيه كما هو).
- **Start Command:** اكتبيه بدقة: **`node server.js`** (وهو الأمر المعرّف في `package.json`).
- **Instance Type / Plan:** اختاري الباقة المجانية **`Free`**.

### 4. إطلاق النشر (Deploy):
- اضغطي في أسفل الصفحة على الزر الأخضر **Create Web Service**.
- سيبدأ Render في بناء ونشر الخدمة فوراً، وسيعرض لك سجلاً (Logs) يقول: `🚀 خادم منصة SAC · SVT prof يعمل بنجاح على المنفذ...`.
- في أعلى الصفحة، ستجدين الرابط المباشر والأخضر لمنصتك: **`https://sac-svt-prof.onrender.com`**! اضغطي عليه وافتحي المنصة لجميع الأساتذة والتلاميذ بفخر! 🎉
