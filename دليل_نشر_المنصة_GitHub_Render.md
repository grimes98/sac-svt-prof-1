# 🚀 دليل نشر منصة SAC · SVT prof على GitHub و Render (مجاناً وخطوة بخطوة)

لقد تم تجهيز مشروعك بالكامل (`sac-svt-app`) وتزويده بخادم **Node.js (`server.js`)** وإعدادات النشر المثالية (`package.json`) ليعمل فوراً وسلسة على منصة **Render (`onrender.com`)** وبدون أي أخطاء.

إليك الخطوات التفصيلية لنشر منصتك وجعلها متاحة لجميع الأساتذة عبر رابط مباشر على الإنترنت:

---

## 📦 المرحلة الأولى: رفع ملفات المشروع على GitHub

### 1. أنشئ مستودعاً جديداً (Repository) على GitHub:
1. اذهب إلى موقع [GitHub.com](https://github.com) وسجل الدخول بحسابك.
2. اضغط على علامة **`+`** في أعلى يمين الشاشة واقتر **New repository**.
3. في خانة **Repository name** اكتب اسماً لمشروعك، مثلاً: `sac-svt-prof-platform` أو `sac-svt-app`.
4. اختر **Public** (عام) ليكون متاحاً لخدمة النشر.
5. اضغط على الزر الأخضر **Create repository**.

### 2. ارفع ملفات المجلد (`sac-svt-app`):
يمكنك الرفع بإحدى الطريقتين:

#### 🔹 الطريقة (أ): السحب والإفلات مباشرة من المتصفح (الأسهل بدون برامج):
1. بعد إنشاء المستودع في GitHub، اضغط على رابط **`uploading an existing file`**.
2. اسحب جميع الملفات والمجلدات الموجودة داخل مجلد `sac-svt-app` (وهي: `index.html`، `admin.html`، `server.js`، `package.json`، ومجلدات `talkhisat/` و `assets/`) وأفلتها في نافذة الرفع.
3. انتظر انتهاء الرفع، ثم اضغط في الأسفل على زر **Commit changes**.

#### 🔹 الطريقة (ب): باستخدام أوامر Git (عبر موجه الأوامر Terminal):
افتح التيرمينال داخل مجلد `sac-svt-app` ونفّذ الأوامر التالية بالترتيب (مع استبدال رابط مستودعك):
```bash
git init
git add .
git commit -m "النسخة الشاملة لمنصة SAC · SVT prof مع لوحة الأدمين وخادم النشر"
git branch -M main
git remote add origin https://github.com/اسم_حسابك/sac-svt-app.git
git push -u origin main
```

## 🌐 المرحلة الثانية: ربط GitHub بـ Render ونشر المنصة (`onrender.com`)

بعد أن أصبحت الملفات على GitHub، انتقل إلى منصة [Render.com](https://render.com) لنشر الموقع مجاناً:

### 1. التسجيل والدخول في Render:
1. ادخل إلى [Render.com](https://render.com) واضغط على **Get Started** أو **Sign in**.
2. اختر التسجيل باستخدام حساب غيت هاب **Sign in with GitHub** لكي يتمكن Render من قراءة مستودعاتك تلقائياً.

### 2. إنشاء استضافة جديدة للمنصة:
بما أننا قمنا ببرمجة وتجهيز خادم `server.js` وملفات الويب معاً، لديك خياران للنشر (كلاهما مجاني وممتاز، وننصح بـ **Static Site** للسرعة الفائقة، أو **Web Service** لتشغيل الخادم):

---

#### ⭐ الخيار الأول (الأسرع والأفضل للمنصات التفاعلية): نشر موقع ثابت (Static Site)
1. في لوحة تحكم Render، اضغط على زر **`New +`** في أعلى اليمين، ثم اختر **`Static Site`**.
2. ستظهر لك قائمة مستودعاتك في GitHub، ابحث عن `sac-svt-app` واضغط على زر **Connect** بجانبه.
3. املأ إعدادات النشر كالتالي:
   * **Name:** اختر اسماً لرابط موقعك، مثلاً: `sac-svt-prof` (سيكون رابطك هو `sac-svt-prof.onrender.com`).
   * **Branch:** `main` (أو `master`).
   * **Build Command:** اتركها فارغة (أو اكتب: `echo "Build OK"`).
   * **Publish Directory:** اكتب `.` (نقطة واحدة تعني المجلد الرئيسي، أو إذا كان مشروعك داخل مجلد فرعي في GitHub اكتب `sac-svt-app`).
4. اضغط في الأسفل على **Create Static Site**.
5. خلال ثوانٍ معدودة، ستكتمل العملية وسيظهر لك الرابط الأخضر المباشر لمنصتك على الإنترنت! 🎉

---

#### ⚙️ الخيار الثاني: نشر خادم ويب Node.js (Web Service)
إذا أردت تشغيل المنصة كخادم Node.js متكامل (باستخدام `server.js` الذي أعددناه):
1. اضغط على زر **`New +`** واختر **`Web Service`**.
2. اربط مستودعك `sac-svt-app` واضغط **Connect**.
3. املأ الإعدادات كالتالي:
   * **Name:** `sac-svt-prof`
   * **Runtime / Environment:** `Node`
   * **Build Command:** `npm install` (أو اتركها فارغة لعدم وجود حزم ثقيلة)
   * **Start Command:** `npm start` (أو `node server.js`)
4. اختر الخطة المجانية **Free ($0/month)**.
5. اضغط على **Create Web Service** وانتظر دقيقة حتى تظهر رسالة `Your service is live` ويصبح موقعك جاهزاً!

## ✅ فحص تجربة الاستخدام بعد النشر:
* **صفحة المنصة المباشرة:** افتح الرابط `https://[your-name].onrender.com` لتصفح الأقسام وتحميل الملخصات.
* **تسجيل الدخول كمسؤول (الأدمين):** اضغط على **تسجيل الدخول** → اختر **دخول المسؤول (Espace admin)** → أدخل الإيميل: `admin@sac-svt.dz` وكلمة المرور: `admin123` لتنتقل فوراً إلى لوحة التحكم الشاملة وإدارة الأساتذة!
